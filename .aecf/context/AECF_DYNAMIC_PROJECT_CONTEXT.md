## Project Overview

**spring-framework-petclinic** is a reference web application demonstrating **Spring Framework 7.0.6** (not Spring Boot) across multiple technology combinations. It manages a veterinary clinic: owners, pets, visits, and vets. The application is packaged as a WAR and targets a Servlet container (Jetty for local dev; Docker via Jib for containerized deployments).

Key characteristics:
- Java 17 (CI also tests Java 21)
- XML-based Spring configuration (no annotation-based component scanning for config)
- Three interchangeable persistence implementations selectable via Spring profile
- JSP/JSTL view layer

## Risk Areas

| Risk | Location | Severity |
|---|---|---|
| Open modification in VCS | `web/OwnerController.java` (git status M) | HIGH вҖ” review before merge |
| No cache TTL | `ClinicServiceImpl.findVets` (@Cacheable "vets") | MEDIUM вҖ” stale data if vets table mutated without restart |
| God service | `ClinicServiceImpl` вҖ” handles all domain operations | MEDIUM вҖ” single point of change for all persistence interactions |
| XML-only config | All Spring beans in XML | MEDIUM вҖ” no compile-time safety; errors surface at container startup |
| JDBC complexity | `JdbcOwnerRepositoryImpl.loadPetsAndVisits` | MEDIUM вҖ” imperative nested mapping, hard to extend |
| Profile parity | 3 implementations of each repository interface | LOW-MEDIUM вҖ” feature/bug divergence risk across profiles |
| No async handling | Entire app is synchronous servlet-based | LOW вҖ” relevant if concurrency requirements increase |
