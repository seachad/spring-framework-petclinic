## Project Overview

**spring-framework-petclinic** is a reference/demo web application built on **Spring Framework 7.0.6** (NOT Spring Boot) showcasing multiple Spring technologies: Spring MVC, Spring Data JPA, Hibernate, and plain Spring JDBC. The app manages a veterinary clinic with owners, pets, visits, and vets.

- **Packaging**: WAR deployed to Servlet container (Jetty for dev; Docker via Jib for container deployment)
- **Java version**: 17 (CI matrix also tests Java 21)
- **Build tool**: Maven 3.8.4+ with Maven Wrapper

## Architecture

**Style**: Modular Monolith — classic layered Spring MVC architecture.

```
┌─────────────────────────────────────────────────────┐
│  Web Layer (Spring MVC)                             │
│  OwnerController, PetController, VetController,     │
│  VisitController, CrashController                   │
│  JSP views in WEB-INF/jsp/                          │
├─────────────────────────────────────────────────────┤
│  Service Layer                                      │
│  ClinicService (interface)                          │
│  ClinicServiceImpl (@Transactional + @Cacheable)    │
├─────────────────────────────────────────────────────┤
│  Repository Layer (3 interchangeable implementations)│
│  profile=jdbc   → repository/jdbc/                 │
│  profile=jpa    → repository/jpa/    (DEFAULT)      │
│  profile=spring-data-jpa → repository/springdatajpa/│
├─────────────────────────────────────────────────────┤
│  Domain Model                                       │
│  Owner, Pet, PetType, Visit, Vet, Specialty         │
└─────────────────────────────────────────────────────┘
```

**Spring Config**: XML-based (`src/main/resources/spring/`)
- `business-config.xml` — repository + service beans, profile switching
- `datasource-config.xml` — DataSource definition
- `mvc-core-config.xml` — DispatcherServlet context
- `mvc-view-config.xml` — view resolver (JSP)
- `tools-config.xml` — AOP, caching (Caffeine), JMX
- `data-access.properties` — JDBC/JPA properties

**Entry Point**: `PetclinicInitializer.java` (Servlet 3.0 programmatic web initializer, replaces web.xml)

## Domain Model

```
BaseEntity (id)
  ├── NamedEntity (name)
  │     ├── PetType
  │     └── Specialty
  └── Person (firstName, lastName)
        ├── Owner (address, city, telephone) ──1:N──► Pet
        └── Vet ──M:N──► Specialty

Pet (birthDate, type, owner) ──1:N──► Visit (date, description)
Vets (XML/JSON wrapper for Vet list)
```
